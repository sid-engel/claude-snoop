# Report Design & Structure

## Cover Page
- Centered logo emoji + tool name
- Report title (--title parameter)
- Target subnet/range (from findings.meta.target)
- Scan timestamp (from findings.meta.timestamp)
- Dark background with white text
- Simple, clean layout

## Executive Summary
Card-style display:
- Hosts Discovered (count from discovery.results)
- Open Ports Found (sum of open_ports across all hosts)
- Vulnerabilities Identified (count from vulns.results)
- Critical Issues (count where severity: "high")

Use accent color for cards.

## Host Discovery Table
**Columns:** IP | Hostname | Status

- All hosts from discovery.results
- Sort by IP ascending
- Hostname shows "—" if not present
- Striped rows (alternate background)

## Port & Service Inventory
**Columns:** IP | Port | Protocol | Service | Product | Version

- All open ports from ports.results[].open_ports
- Sort by IP, then port ascending
- One row per open port
- Striped rows

## Vulnerabilities & Findings
**Columns:** IP | Port | Product | Version | CVE/Finding | Severity | Description

- All entries from vulns.results
- Each severity level gets badge (high: red, medium: orange, low: blue, informational: purple, update_available: green)
- Sort by severity (high → medium → low → informational), then by IP
- If no vulnerabilities: "No vulnerabilities detected"
- Include both CVE entries and update_available entries

## Footer
- "Generated by claude-snoop"
- Timestamp of report generation
- Target that was scanned

## Page Style
- Use primary color for headings
- Use secondary color for table headers
- Use background color for page background
- Use border color for table borders
- Maintain consistent spacing per design.spacing
- Use font family/sizes per design.fonts
